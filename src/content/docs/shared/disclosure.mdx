import { Disclosure } from '@interledger/docs-design-system'
import { Tabs, TabItem } from '@astrojs/starlight/components'

:::caution
In October 2024, we decided to move away from using the Disclosure component in favor of the standard HTML [details element](/classes/details). Please use `<details>` going forward. We may deprecate the Disclosure component in the future.
:::

This component allows you to show/hide content within a collapsible container.

To use the component, the page must be in `.mdx` format. Change the format from `.md` to `.mdx` if necessary. All your existing markdown will still be supported without issue.

## Usage

Import the `Disclosure` component like so:

```jsx
import { Disclosure } from '@interledger/docs-design-system'
```

Use the `<Disclosure>` component within your content like so:

```jsx
<Disclosure toggleText="Show code snippets">
  {/* Your content, can be markup or markdown */}
</Disclosure>
```

:::note[Single vs double quotes]
In general, the `toggleText` can be wrapped in single or double quotes. However, if the content contains punctuation, you must wrap the content in double quotes to avoid errors.
:::

You can nest the Disclosure component with the built-in Starlight [`<Tabs>`](https://starlight.astro.build/guides/components#tabs) component. This is helpful when displaying multiple related code snippets.

````jsx
import { Tabs, TabItem } from '@astrojs/starlight/components'

<Disclosure toggleText="Show code snippets">
   <Tabs>
      <TabItem label='Request'>
      ```bash
      GET /alice HTTP/1.1
      Accept: application/json
      Host: wallet.example
      ```
      </TabItem>
      <TabItem label='Response'>
      ```bash
      HTTP/1.1 200 Success
      Content-Type: application/json

      {
         "id":"https://wallet.example/alice",
         "assetCode":"USD",
         "assetScale":2,
         "authServer":"https://wallet.example/auth",
      }
      ```
      </TabItem>

   </Tabs>
</Disclosure>
````
